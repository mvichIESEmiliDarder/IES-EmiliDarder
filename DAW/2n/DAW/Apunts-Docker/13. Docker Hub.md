
## ☁️ Docker Hub: El Registre de Contenidors

### Diapositiva 1: Què és Docker Hub?

**Títol:** Docker Hub: El Cor de la Col·laboració en Contenidors

  * **Definició:**

      * **Docker Hub** és el **registre de contenidors públic i privat** més gran del món.
      * És un servei centralitzat on s'emmagatzemen i es distribueixen les imatges de Docker.

  * **Funció Principal:**

      * Actua com un **repositori** per a imatges: proporciona la font principal d'imatges de sistemes operatius, aplicacions i eines, tant oficials com de la comunitat.
      * Facilita la **col·laboració** en permetre als usuaris compartir les seves pròpies imatges amb la resta del món.

  * **Com Funciona (Automàticament):**

      * Quan executes una imatge amb `docker run` que no tens localment (p. ex., `docker run rocker/verse`), Docker comprova si existeix a l'ordinador.
      * Si no hi és, **automàticament la descarrega (pulls)** des de Docker Hub.

-----

### Diapositiva 2: Obtenció i Descarrega d'Imatges (Pulling)

**Títol:** Com Obtenir Imatges del Hub

  * **Descarrega Implícita:**

      * La descarrega es produeix automàticament quan executes una imatge inexistent localment:
        ```bash
        docker run --rm -p 8787:8787 rocker/verse 
        ```

  * **Descarrega Explícita:**

      * Si només vols baixar la imatge al teu ordinador sense executar-la immediatament, utilitza la comanda `docker pull`:
        ```bash
        docker pull rocker/verse
        ```

  * **Estructura de la Imatge:**

      * Les imatges es refereixen típicament amb el format **`usuari/nom_repositori:etiqueta`** (o només `nom_repositori:etiqueta` per a imatges oficials).

-----

### Diapositiva 3: Compartir Imatges Personalitzades (Pushing)

Si has creat la teva pròpia imatge i vols compartir-la, has de seguir aquests passos per pujar-la a un repositori de Docker Hub:

1.  **Crear un Compte:** Registra't a [hub.docker.com](https://hub.docker.com/).
2.  **Crear un Repositori:** Inicia sessió al lloc web i crea un repositori nou (p. ex., `verse_gapminder`).
3.  **Iniciar Sessió Local:** Autentica't des de la línia de comandes:
    ```bash
    docker login --username=el_teu_usuari_hub
    # Introdueix la teva contrasenya quan se et demani.
    ```
4.  **Etiquetar la Imatge (Tagging):** Assigna un nom i una etiqueta (tag) a la teva imatge local perquè coincideixi amb el format del teu repositori a Docker Hub. Has de conèixer l'**ID de la Imatge** (obtingut amb `docker images`).
    ```bash
    docker tag [ID_IMATGE] el_teu_usuari_hub/nom_repositori:etiqueta
    # Exemple: docker tag bb38976d03cf el_teu_usuari/verse_gapminder:v1.0
    ```
5.  **Pujar la Imatge (Pushing):** Puja la imatge etiquetada al repositori:
    ```bash
    docker push el_teu_usuari_hub/nom_repositori
    ```
    *Un cop fet, la teva imatge estarà disponible públicament (o privadament, segons la configuració del repositori) per ser descarregada per qualsevol.*

### Diapositiva 4: Alternatives Locals: Guardar i Carregar Imatges

En alguns casos, és útil guardar i carregar imatges localment, especialment per evitar problemes d'ample de banda o si es vol utilitzar la imatge en entorns sense accés directe a Docker Hub.

  * **Guardar una Imatge (`docker save`):**

      * Converteix la imatge de Docker en un **arxiu tar** per a emmagatzematge o transferència local.

    <!-- end list -->

    ```bash
    docker save nom_imatge > nom_imatge.tar
    # Exemple: docker save verse_gapminder > verse_gapminder.tar
    ```

  * **Carregar una Imatge (`docker load`):**

      * Reconstrueix la imatge de Docker a partir d'un arxiu tar prèviament guardat.

    <!-- end list -->

    ```bash
    docker load --input nom_imatge.tar
    ```

### Diapositiva 5: Comandes Clau de Docker Hub

| Comanda | Descripció | Ús (Exemple) |
| :--- | :--- | :--- |
| **`docker login`** | Inicia sessió a Docker Hub des de la terminal. | `docker login` |
| **`docker pull`** | Descarrega una imatge del Hub al sistema local. | `docker pull nginx:latest` |
| **`docker images`** | Llista les imatges locals, incloent-hi els seus IDs. | `docker images` |
| **`docker tag`** | Etiqueta una imatge local per preparar-la per a la pujada. | `docker tag ID_IMATGE usuari/repo:tag` |
| **`docker push`** | Puja la imatge etiquetada al repositori de Docker Hub. | `docker push usuari/repo` |
| **`docker save`** | Exporta una imatge local a un fitxer `.tar`. | `docker save imatge > imatge.tar` |
| **`docker load`** | Importa una imatge des d'un fitxer `.tar`. | `docker load --input imatge.tar` |
